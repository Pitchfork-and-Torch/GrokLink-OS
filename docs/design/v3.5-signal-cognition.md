# GrokLink OS v3.5 — Signal Cognition

**Status:** Implemented (host complete; firmware light-stats path landed)  
**Baseline:** v3.4.0 Signal World Exploration Report (2026-07-22)  
**Safety:** Passive sensory only · no TX · no third-party decode · edu-gated · default-deny actuators  

---

## STEP 0 — Report internalization (confirmed)

The v3.4.0 report establishes:

- Live silicon: DFU → CDC GrokRPC → edu → light RX `sim:false` (edges + RSSI only).
- 16 OpenAI tools; schema `groklink.signal_observation.v1`.
- Strengths: end-to-end ownership, honest sim flags, ROM missions without SD, passive tool surface.
- Explicit limitations: light RX only, occupancy is host heuristic, no protocol decode, Windows CDC quirks, serial desync under long bursts.
- Open roadmap: Full SD FatFs (deferred; not this phase).
- Conclusion: next work = **calibrated spectrum tooling, richer event taxonomy, host monitor UX** — not weakening default-deny.

### Winner: **v3.5 Signal Cognition** (candidate 1)

**Why highest leverage + integrity**

1. Directly attacks the report’s #1 agent-facing gap: occupancy without calibration looks “high” even for noise-floor edge chatter.
2. Stays a **pure passive RF sensory peripheral** — no decode, no TX.
3. Mostly host intelligence + tiny additive device stats → fits **heap_free ≈ 4096**.
4. Additive schema/tools → existing 16 tools keep working.
5. Unlocks better monitor UX and missions later without redoing the sensory contract.

## STEP 2 — Detailed plan

### Vision

> v3.5 makes every observation **relative to a measured noise floor and pulse-rate baseline**, with a richer **non-decode event taxonomy**, so agents narrate *change* and *elevation* instead of inventing protocols.

### Workstreams

1. **Schema** — `groklink.signal_observation.v2` additive; v1 fields preserved.
2. **Host bridge** — calibration store, packager stats, new tools, CLI, tests.
3. **Firmware** — light-RX pulse_rate + dual RSSI samples; RPC JSON fields; ROM `lab_noise_baseline`.
4. **Offline agent** — optional use of pulse_rate in vault narratives (host-side primarily).
5. **GUI** — optional last pulse_rate on RADIO page (minimal).
6. **Docs / skill** — playbook, exploration extension, skill scripts.
7. **Tests** — host unit tests without hardware.

### Milestones

| M | Deliverable | Acceptance |
|---|-------------|------------|
| M1 | Schema v2 + JSON schema file | v1 observations still validate; v2 fields optional |
| M2 | Packager calibration + taxonomy | Quiet/high samples produce calibrated_occupancy + events |
| M3 | Tools: taxonomy, noise floor, calibration, compare | Mock client tests green |
| M4 | Firmware stats + RPC | DFU builds; fields present when device has them |
| M5 | Docs + skill + VERSION 3.5.0 | Public hygiene push |

### Non-goals (safety boundary)

- No TX / GPIO / system automation.
- No third-party protocol decode or payload_hex population.
- No removal of edu gate or confirm tokens.
- No FatFs / SD hot-load (still roadmap).
- No multi-device mesh.

### Heap budget

- Device: only a few `int32`/`int16` fields on stack result structs (no large buffers).
- Host: rolling calibration map keyed by band_label (≤8 bands, ≤256 B).

### Backward compatibility

- Existing 16 tools unchanged in name/behavior.
- Schema id becomes v2 but packager still fills all v1 fields.
- `schema_compat: ["groklink.signal_observation.v1"]` advertised.
- Raw RPC gains optional keys; old hosts ignore them.

## STEP 3 — Design summary

### Schema v2 additions

- `calibration`: noise_floor_dbm, baseline_pulse_rate_hz, snr_est_db, method, age_ms
- `stats`: pulse_rate_hz, rssi_min_dbm, rssi_max_dbm, dwell_ms
- `activity.calibrated_occupancy`: quiet|ambient|elevated|busy|unknown
- `activity.confidence`: 0.0–1.0
- `events[]` types: `quiet_dwell`, `edge_activity`, `elevated_vs_noise`, `band_hotspot`, `noise_floor_sample` (never `decoded_frame`)
- `safety.tx` always false; `policy_context.passive_only` always true

### New tools (additive → 20 total)

| Tool | Passive |
|------|------|
| `get_event_taxonomy` | Document allowed event types (no decode) |
| `get_calibration_state` | Host baseline map |
| `observe_noise_floor` | Short RX → store baseline for band |
| `observe_compare` | Two-frequency passive compare |

### Data flow (text)

**observe_rx:**  
LLM → tool → edu_ack → `subghz_rx` → raw edges/RSSI/stats → packager(+calibration) → v2 observation → store → narrative

**spectrum:**  
LLM → tool → sequential RX → packager spectrum + hottest + per-band calibrated_occupancy → observation

**monitor:**  
start_monitor → loop short RX → chunk packager → get_monitor_chunk; baselines optional from calibration store

### Safety proof sketch

Every new tool path only calls `subghz_rx` / status / read APIs. No `subghz_tx`, no confirm mint for actuators. Packager hardcodes `safety.tx=false`. Event types exclude payload fields.
